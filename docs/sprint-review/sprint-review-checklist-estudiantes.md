# Checklist - Sprint Review (estudiantes)

Usar antes de entregar el video del sprint.

## Antes de grabar

- [ ] Definimos PO, SM y responsabilidades del equipo.
- [ ] Revisamos objetivos comprometidos del sprint.
- [ ] Reunimos evidencias (issues, ramas, PRs, docs, demo tecnica).
- [ ] Acordamos el orden de exposicion de todos los integrantes.

## Durante el video

- [ ] Nos presentamos como equipo y decimos el numero de sprint.
- [ ] Indicamos quien fue PO, SM y dev team.
- [ ] Cada integrante explica que hizo y muestra evidencia.
- [ ] Mostramos una demo real del incremento del sprint.
- [ ] Explicamos que salio bien y que se debe mantener.
- [ ] Explicamos que salio mal y por que.
- [ ] Definimos mejoras concretas para el siguiente sprint.

## Antes de enviar

- [ ] Verificamos duracion sugerida segun sprint.
- [ ] El audio/video se entiende correctamente.
- [ ] El link al video es publico y accesible.
- [ ] Incluimos el link en el README o documento de entrega.
- [ ] Confirmamos que participaron todos los integrantes.
