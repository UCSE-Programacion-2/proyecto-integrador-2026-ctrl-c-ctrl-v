# Frontend

## Objetivo
Implementar la interfaz de usuario y consumir la API del backend.

## Recomendación docente: comparar dos versiones

Este proyecto incluye dos implementaciones para que, al final del año, puedan comparar enfoques y madurez del código:

- `frontend-vanilla/`: sin framework (HTML/CSS/JS).
- `frontend-react/`: con React.

## Consigna general (obligatoria en ambas)

1. Implementar los mismos flujos que el backend expone (mismas pantallas y operaciones).
2. Consumir los mismos endpoints (mismo “contrato” de la API).
3. Mantener un comportamiento equivalente en:
   - carga (loading)
   - errores (mensajes)
   - estados vacíos (sin datos)

## Estructura

- `frontend-vanilla/`: documentación y guía para la versión vanilla.
- `frontend-react/`: documentación y guía para la versión React.

Para los requisitos por sprint, usen como referencia:

- `docs/sprint-1/`
- `docs/sprint-3/`
- `docs/sprint-4/`

## Checklist final antes de entregar

- [ ] Ambos frontends muestran los mismos flujos principales.
- [ ] La navegación está completa (no hay pantallas “sueltas”).
- [ ] Se controla el error de red y errores del backend.
- [ ] Hay un archivo README en cada implementación con cómo ejecutar.
